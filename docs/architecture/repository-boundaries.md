# Repository Data Boundaries

This document reserves locations without defining their future data contracts.

- `data/canonical-records/` will hold canonical editable records.
- `data/publication-snapshots/` will hold immutable publication snapshots.
- `generated/release-data/` will hold generated release data.
- `dist/` holds generated static output from Astro.

The later owning tickets will define formats, lifecycle rules, and tracking policy. These directories remain absent until those tickets create real content or generated output.
